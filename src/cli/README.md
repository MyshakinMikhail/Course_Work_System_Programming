# cli

Модуль запускает приложение и связывает пользовательский ввод с цепочкой `Parser -> Engine`.

## Режимы запуска

Интерактивный режим:

```bash
./prog
```

Пакетный режим:

```bash
./prog script.sql
```

## Состав

- `main.cpp` - точка входа и выбор режима запуска;
- `CliApp` - общий сценарий выполнения команд;
- `CommandReader` - чтение команд до символа `;`.

## Поведение

- команды могут занимать несколько строк;
- символ `;` внутри строкового литерала не завершает команду;
- успешные сообщения и JSON выводятся в `stdout`;
- ошибки выводятся в `stderr`;
- после ошибки выполнение batch-файла продолжается;
- приложение не должно аварийно завершаться из-за ошибок парсинга или исполнения.

## Пример script.sql

```sql
CREATE DATABASE app;
USE app;
CREATE TABLE users (id int INDEXED, name string);
INSERT INTO users (id, name) VALUE (1, "Ann"), (2, "Bob");
SELECT * FROM users WHERE id >= 1;
```

# engine

Модуль выполняет команды, полученные от `parser`, и является границей между SQL-слоем и `core`/`storage`.

## Состав

- `Engine` - принимает `ParsedCommand` и диспетчеризует выполнение;
- `ExecutionResult` - единый результат выполнения команды;
- `EngineError` - ошибка семантики или исполнения;
- `DatabaseCatalog` - каталог баз данных и таблиц в файловой системе;
- `ConditionEvaluator` - вычисление `WHERE`;
- `JsonFormatter` - форматирование результата `SELECT` в JSON через `nlohmann/json`.

## Хранение

По умолчанию данные пишутся в директорию `data`:

```text
data/
  app/
    users.tbl
```

Имя `users` резолвится через активную базу данных, выбранную командой `USE`. Имя `app.users` использует явно указанную базу данных.

## Реализовано

- `CREATE DATABASE`;
- `DROP DATABASE`;
- `USE`;
- `CREATE TABLE`;
- `DROP TABLE`;
- `INSERT`;
- `SELECT`;
- `SELECT ... WHERE` с сравнениями, `BETWEEN`, `LIKE`;
- JSON-вывод строк выборки.

## Ограничения

- `UPDATE` и `DELETE` пока возвращают заглушку: текущий `core/storage` не поддерживает перезапись и удаление записей.
- `NULL` парсится, но не сохраняется: текущий `Field` не имеет nullable-состояния.
- `NOT_NULL` парсится, но не сохраняется в `TableSchema`, потому что в `Column` пока нет такого поля.
- `INDEXED` сохраняется в схеме, но уникальность и оптимизация через индекс зависят от реализации нижнего слоя `index/core`.
- `INSERT` временно требует значения для всех колонок.

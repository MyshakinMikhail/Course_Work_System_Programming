# storage

Модуль `storage` отвечает за бинарное хранение таблиц в одном файле по схеме `[SCHEMA][DATA]` и предоставляет слой чтения и записи для `Table`.

## Формат файла

1. `SCHEMA`
   - количество колонок;
   - для каждой колонки:
     - имя;
     - тип (`INT` или `STRING`);
     - признак `indexed`.
2. `DATA`
   - записи таблицы пишутся последовательно;
   - каждая запись хранит число полей и значения полей в порядке схемы.

## FileManager

`FileManager` реализует:

- `writeSchema(TableSchema schema)` - записывает схему в начало файла;
- `readSchema()` - читает схему из файла;
- `writeRecord(Record record)` - дописывает запись в конец файла и возвращает ее `offset`;
- `readRecord(offset)` - читает запись по сохраненному `offset`;
- `readAllRecords()` - последовательно читает все записи из секции `DATA`.

## Валидация

`FileManager` дополнительно проверяет:

- имена колонок в схеме не пустые;
- типы колонок и полей ограничены `INT` и `STRING`;
- путь таблицы указывает на обычный файл;
- файл таблицы существует перед чтением схемы;
- количество и типы полей записи совпадают со схемой.

## Offset

`offset` вычисляется через стандартные файловые позиции:

- `tellp()` - после перехода в конец файла перед записью;
- `tellg()` - после чтения схемы, чтобы запомнить начало секции `DATA`.

Это позволяет внешнему индексу ссылаться на конкретную запись без дополнительного сканирования файла.
